Class: Xhr

Xhr

This class is a helper which makes the handling of XML-Http-Request's a bit easier. The class adds one level of abstraction when providing access to the browsers native XHR object (x-browser-compatible). This enables us also to access both the Dom and the Xhr with the same code (interface) and this saves us a lot of extra code in core.

create(requestObject) → {Lazyload.Pipe.Xhr}

Creates and preconfigures a new XML-HTTP-REQUEST-object with the passed in configuration in requestObject. To tell the XHR that it should start with loading of the resource you have to call the open()-method after the call to create(). The calls can be chained: e.g. new Xhr().create({}).open();

<private, inner> _getXhr() → {Object}

This method is intend to create an instance of the browser's native (i call it XML-HTTP-REQUEST-object) object. This is of course done in a x-browser-compatible way. I decided here to support IE5.5 too (currently not fully implemented) cause it's not that much more work and Lazyload.js will cover up to 100% of the (modern) Browsers out there.
