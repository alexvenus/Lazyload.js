Class: Annotation

Annotation

The Annotation-class provides the functionality for parsing out the dependencies of a JavaScript sourcefile. It identifies the dependencies by searching for the javadoc style annotations. Lazyload.js currently makes use of the following annotations: @resources, @configuration, @properties (see example) With @resources you define dependencies of the current code-block (a code-block is any part of code with a javadoc-style comment. Within one single physical file can exist n parts of Lazyload.js directives) Use @resources like this: @resources path/to/file/foo.js (single file) or @resources path/to/file/foo.js, //ajax.googleapis.com/ajax/libs/jquery/1.8.3/jquery (multiple resources - loaded in order they appear). With @configuration you can define a configuration object which is merged with existing configuration when loading resources from @resources. The passed "@configuration" will then overwrite values in the existing configuration . With @properties you can define a property object which is merged with the fresh created lazyObject skeleton. The passed @properties overwrites existing values.
