It's just about loading files smart and easily!

require('foo.js', 'bar.js').and('baz.js').then(success, error);

Lazyload.js has a minified size of just ~6.6 Kb. Despite the small size Lazyload.js provides an amazing set of features like: serial and parallel loading (js, css, bin), caching (session, localStorage), managing annotation based dependencies and so many more. Lazyload.js runs in all modern browsers (Firefox, Chrome, Safari, Opera, ...) and - surprise - in Internet Explorer (6 - 10), too!

  • With Lazyload.js is loading files simple like never before. Load files in the required order - without hazzle - no matter if serial or parallel.

  • Use Lazyload.js with basket.js - Lazyload.js provides a perfect interface for external loader and supports loading with basket.js (for caching in localStorage) native.

Lazyload is stable and well tested in many modern and older browser. Lazyload is developed using grunt as buildsystem and so everything get lint, tested (QUnit) and bundled for you ;) Current build status:

Lazyload.js is fast, does not depend on any other JavaScript framework and has a clean, obvious syntax so that you can begin using it so easily.

Features

  • Loading of JavaScript-, CSS- or Binary-files

  • Parallel and serial loading of JS, CSS, Binary-Files

  • Caching (storing to / loading from localStorage)

  • Annotation based dependency-management (e.g. @resources)

Loading of JavaScript-, CSS- or Binary-files

Lazyload.js makes the loading of resources easy like never before. There are some similar libraries out there but i didn't liked their API and the size of them was often round about triple the size of Lazyload.js. Give Lazyload.js a try! It's the perfect tool for loading resources like JavaScript (adding JavaScript files with dependency trees of any size at runtime to DOM), CSS (adding stylesheets at runtime to DOM and apply them to all matched elements), Binary-files - prefetch data like images, swf's ... intelligent with Lazyload.js.

Parallel and serial loading of JavaScript-, CSS-, Binary-Files

With Lazyload.js it's possible to load files both, asynchronous (parallel) and synchronous (serial) depending on your needs. Lazyload uses XHR-pipes to load the content of a file into the active scope (DOM). If you want to load files synchronously (one after another - maybe to handling dependencies manually) you can make use of the nice function and() and combine it with the existing call e.g. like this: Lazyload.require('jquery.js').and('jquery-ui.js').then(...); In fact: parallel (asynchronous) and serial (synchronous) loading of resources is simple as never before.

Caching (storing to / loading from localStorage)

Lazyload.js feat. basket.js (a nice library from Addy Osmani). This is currently my favorite new feature! I've implemented a switch in __fetchFileFromRemote()_ and made it possible to configure Lazyload.js to make use of an external loader like basket.js. I've found the link to basket.js on Twitter a few days ago and after trying out basket.js i decided to feat. this project cause it does a good job (see Example 5 - Load and configure basket.js) and makes Lazyload.js even more useful.

Annotation based dependency-management (e.g. @resource)

Another rocking feature: All you need to do, to make use of this feature is using the @-annotation "@resources" in the JavaScript source and Lazyload.js does the dirty job of handling those dependencies. It loads all required files in the correct order into the scope (DOM). No matter how deep the dependencies are nested or if circular dependencies are found. Lazyload.js uses nested-sets and is fast and smart at the same time while managing your project's dependencies.

I Can Has Code?

Here an ordered, but not fully complete list of examples. Just a short demonstration of the usage. Take this as an inspiration to build great things :)

  1. Loading a single file

    Lazyload.require('my/script/jquery-1.8.3.js');

  2. Loading n files parallel (async)

    Lazyload.require(
'my/script/jquery-1.8.3.js, my/script/jquery-ui.js'
);

  3. Loading n files - but load jquery-ui right after jquery-1.8.3.js is loaded

    Lazyload.require(
'my/script/jquery-1.8.3.js'
).and(
'my/script/jquery-ui.js'
);

  4. Loading n files - but load jquery-ui right after jquery-1.8.3.js is loaded - but this time we use annotations in jquery-ui.js which will effect in loading jquery-1.8.3.js first

    Lazyload.require('my/script/jquery-ui.js');

  5. Loading basket.js first and afterwards configure Lazyload to use basket.js as loader and load jquery

    // we require basket cause we want Lazyload to cooperate 
// with basket for localStorage API access.
Lazyload.require(
'my/script/basket.js'
).then(
function success(lazyobject){
    // configure Lazyload to use basket for storage
    Lazyload.config({
        loader: window.basket
    });

    // now start loading all the stuff which is stored
    // locally in localStorage (if available)
    Lazyload.require(
        'my/script/jquery-1.8.3.js'
    );
}
,
function failure(failure){
    alert(failure);
}
);

Used in the wild?

A very early prototype was and is still used in a large scaled backoffice Web-Application from Telekom Deutschland GmbH and runs stable for over 3 years now. So the codebase should be almost ready for real world applications.

Any Question?

Maybe this is the answer: Take Lazyload.js, include it in your site, call require(file[s]) first, then chain as many and(file[s]) calls after another and finally but also optionally add a call to then(success, failure) with your success and/or failure callback(s). That's it!
